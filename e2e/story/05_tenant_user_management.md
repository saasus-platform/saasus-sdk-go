# 05. テナントユーザー管理ストーリー

## 概要
テナントに所属するユーザーの管理、役割割り当てを行うE2Eテストストーリー

## テスト対象API
- tenantUser.yml

## ストーリー

### 1. テナントユーザーの基本管理
- [ ] **GET /tenants/{tenant_id}/users** - テナントユーザー一覧を取得
  - 特定テナントに所属するユーザー一覧の取得
  - ユーザーの環境・役割情報の確認
- [ ] **POST /tenants/{tenant_id}/users** - テナントユーザーを作成
  - テナントに新規ユーザーを作成
  - 属性情報の設定
- [ ] **GET /tenants/{tenant_id}/users/{user_id}** - テナントユーザー詳細を取得
  - 特定ユーザーの詳細情報取得
  - 環境毎の役割情報確認
- [ ] **PATCH /tenants/{tenant_id}/users/{user_id}** - テナントユーザー属性を更新
  - ユーザーの追加属性情報を更新
- [ ] **DELETE /tenants/{tenant_id}/users/{user_id}** - テナントユーザーを削除
  - テナントからユーザーを削除

### 2. 全テナントユーザーの管理
- [ ] **GET /tenants/all/users** - 全テナントユーザー一覧を取得
  - 全テナントに所属するユーザーの一覧取得
  - 複数テナント所属ユーザーの確認
- [ ] **GET /tenants/all/users/{user_id}** - 全テナントでのユーザー情報を取得
  - ユーザーIDから全テナントでの所属情報を取得
  - 複数テナントでの役割確認

### 3. テナントユーザーの役割管理
- [ ] **POST /tenants/{tenant_id}/users/{user_id}/envs/{env_id}/roles** - ユーザー役割を作成
  - 特定環境でのユーザーに役割を付与
  - 複数役割の同時付与
- [ ] **DELETE /tenants/{tenant_id}/users/{user_id}/envs/{env_id}/roles/{role_name}** - ユーザー役割を削除
  - 特定環境でのユーザーから役割を削除
  - 役割削除後の権限確認

## テストシナリオ例

### シナリオ1: 新規テナントユーザーの作成と役割付与
1. テナントに新規ユーザーを作成
2. 作成したユーザーに管理者役割を付与
3. ユーザー情報を取得して役割が正しく設定されていることを確認
4. 別の環境で一般ユーザー役割を付与
5. 環境毎の役割設定を確認

### シナリオ2: 複数テナント所属ユーザーの管理
1. 既存ユーザーを別のテナントに追加
2. 各テナントで異なる役割を付与
3. 全テナントユーザー一覧でユーザーが複数回表示されることを確認
4. ユーザーIDから全テナント情報を取得
5. テナント毎の役割の違いを確認

### シナリオ3: ユーザー属性の管理
1. テナントユーザーの属性情報を更新
2. 更新された属性情報を確認
3. 属性情報の削除・再設定
4. 属性情報の整合性確認

### シナリオ4: 役割の動的変更
1. ユーザーに複数の役割を付与
2. 特定の役割を削除
3. 残った役割の確認
4. 新しい役割の追加
5. 最終的な役割設定の確認

## 前提条件
- テナントが作成されている
- 役割(ロール)が定義されている
- 環境が設定されている
- SaaSユーザーが存在している

## 期待結果
- テナントユーザーが正常に作成・管理できる
- 役割の付与・削除が正常に動作する
- 環境毎の役割設定が機能する
- 複数テナント所属ユーザーが適切に管理される
- ユーザー属性が正しく更新される

## 注意事項
- ユーザーIDは一意だが、テナント毎に異なるオブジェクトとして扱われる
- 役割の変更は即座に認可に影響する
- テナントからユーザーを削除してもSaaSユーザー自体は削除されない
