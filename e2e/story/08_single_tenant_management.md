# 08. シングルテナント管理ストーリー

## 概要
シングルテナント機能の設定と管理を行うE2Eテストストーリー

## テスト対象API
- singleTenant.yml

## ストーリー

### 1. シングルテナント設定の管理
- [ ] **GET /single-tenant/settings** - シングルテナント設定を取得
  - 現在の設定状況確認
  - 有効/無効状態の確認
  - CloudFormationテンプレートURL確認
  - DDLテンプレートURL確認
  - AssumeRole用のロールARN確認
  - ExternalID確認
- [ ] **PATCH /single-tenant/settings** - シングルテナント設定を更新
  - シングルテナント機能の有効化
  - CloudFormationテンプレートファイルのアップロード
  - DDLテンプレートファイルのアップロード
  - AssumeRole用ロールARNの設定
  - ExternalIDの設定

### 2. CloudFormationスタック管理
- [ ] **GET /single-tenant/cloudformation-launch-stack-link** - CloudFormationスタック起動リンクを取得
  - スタック作成用のリンク取得
  - リンクの有効性確認
  - パラメータの確認

## テストシナリオ例

### シナリオ1: シングルテナント機能の初期設定
1. 現在のシングルテナント設定を取得
2. 機能が無効になっていることを確認
3. CloudFormationテンプレートを準備
4. DDLテンプレートを準備
5. AWS IAMロールを作成（テスト環境）
6. シングルテナント設定を更新して機能を有効化
7. 設定が正しく保存されていることを確認

### シナリオ2: CloudFormationテンプレートの管理
1. Base64エンコードしたCloudFormationテンプレートで設定更新
2. 設定取得でテンプレートURLが生成されていることを確認
3. テンプレートファイルの更新
4. 新しいテンプレートURLが生成されることを確認

### シナリオ3: DDLテンプレートの管理
1. Base64エンコードしたDDLテンプレートで設定更新
2. 設定取得でDDLテンプレートURLが生成されていることを確認
3. DDLテンプレートファイルの更新
4. 新しいDDLテンプレートURLが生成されることを確認

### シナリオ4: AWS連携設定
1. AssumeRole用のIAMロールARNを設定
2. ExternalIDを設定
3. 設定が正しく保存されていることを確認
4. CloudFormationスタック起動リンクを取得
5. リンクにロール情報が含まれていることを確認

### シナリオ5: 設定の無効化
1. シングルテナント機能を無効化
2. 設定が無効になっていることを確認
3. CloudFormationスタック起動リンクの取得エラー確認

### シナリオ6: エラーハンドリング
1. 無効なロールARNで設定更新を試行（400エラー確認）
2. 不正なBase64データでテンプレート更新を試行（エラー確認）
3. 機能が無効な状態でスタック起動リンク取得を試行（エラー確認）

## 前提条件
- 基本セットアップが完了している
- AWS IAMロールが作成されている（テスト環境）
- CloudFormationテンプレートが準備されている
- DDLテンプレートが準備されている
- 適切な権限でAPIにアクセスできる

## 期待結果
- シングルテナント設定が正常に管理できる
- CloudFormationテンプレートが正しくアップロードされる
- DDLテンプレートが正しくアップロードされる
- AWS連携設定が正常に動作する
- CloudFormationスタック起動リンクが正しく生成される
- 適切なエラーハンドリングが行われる

## 注意事項
- シングルテナント機能の有効化には特定の条件が必要な場合がある
- CloudFormationテンプレートは有効なAWSリソース定義である必要がある
- AssumeRole用のIAMロールには適切な権限が必要
- ExternalIDはセキュリティのため適切に管理する
- テンプレートファイルはBase64エンコードして送信する

## セキュリティ考慮事項
- IAMロールの権限は最小限に設定する
- ExternalIDは推測困難な値を使用する
- CloudFormationテンプレートに機密情報を含めない
- テスト後は不要なAWSリソースを削除する
