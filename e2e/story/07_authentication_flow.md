# 07. 認証フローストーリー

## 概要
認証・認可情報の取得と管理を行うE2Eテストストーリー

## テスト対象API
- credential.yml

## ストーリー

### 1. 認証・認可情報の保存と取得
- [ ] **POST /credentials** - 認証・認可情報を保存
  - IDトークン、アクセストークン、リフレッシュトークンの保存
  - 一時コードの生成
  - 一時コードの有効期限確認（10秒）
- [ ] **GET /credentials** - 認証・認可情報を取得
  - 一時コード（tempCodeAuth）による認証情報取得
  - リフレッシュトークン（refreshTokenAuth）による認証情報取得
  - 無効な一時コードでのエラー確認
  - 期限切れ一時コードでのエラー確認

## テストシナリオ例

### シナリオ1: 一時コード認証フロー
1. 有効なIDトークン、アクセストークン、リフレッシュトークンで認証情報を保存
2. 返却された一時コードを確認
3. 一時コード（tempCodeAuth）で認証情報を取得
4. 取得した認証情報が保存時と一致することを確認
5. 同じ一時コードで再度取得を試行（エラー確認）

### シナリオ2: リフレッシュトークン認証フロー
1. 認証情報を保存してリフレッシュトークンを取得
2. リフレッシュトークン（refreshTokenAuth）で新しい認証情報を取得
3. 新しいアクセストークンが取得できることを確認
4. 無効なリフレッシュトークンでのエラー確認

### シナリオ3: 一時コードの有効期限テスト
1. 認証情報を保存して一時コードを取得
2. 10秒以内に一時コードで認証情報を取得（成功確認）
3. 新しい一時コードを生成
4. 10秒以上待機
5. 期限切れ一時コードで認証情報取得を試行（エラー確認）

### シナリオ4: 無効なパラメータでのエラーハンドリング
1. 無効なIDトークンで認証情報保存を試行（400エラー確認）
2. 存在しない一時コードで認証情報取得を試行（404エラー確認）
3. 無効なリフレッシュトークンで認証情報取得を試行（エラー確認）
4. 必須パラメータ不足での保存試行（400エラー確認）

### シナリオ5: 認証フローの切り替え
1. 認証情報を保存
2. デフォルト（tempCodeAuth）で認証情報を取得
3. auth-flowパラメータを明示的にtempCodeAuthに設定して取得
4. リフレッシュトークンを使用してrefreshTokenAuthで取得
5. 各フローで適切な認証情報が取得できることを確認

## 前提条件
- 基本セットアップが完了している
- 有効なIDトークン、アクセストークン、リフレッシュトークンが準備されている
- 認証設定が適切に行われている

## 期待結果
- 認証・認可情報が正常に保存できる
- 一時コードが正しく生成される
- 一時コードによる認証情報取得が機能する
- リフレッシュトークンによる認証情報取得が機能する
- 一時コードの有効期限が正しく管理される
- 適切なエラーハンドリングが行われる

## 注意事項
- 一時コードの有効期限は10秒と短いため、テストタイミングに注意
- 一時コードは一度使用すると無効になる
- リフレッシュトークンの有効期限は設定により異なる
- 認証情報は機密情報のため、テスト後の適切な削除が必要

## セキュリティ考慮事項
- テスト用の認証情報は本番環境のものを使用しない
- 一時コードの短い有効期限によりセキュリティが確保される
- 認証情報の保存・取得ログを適切に管理する
