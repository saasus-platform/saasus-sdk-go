# 01. 基本セットアップストーリー

## 概要
SaaSus Auth APIの基本的なセットアップと設定を行うE2Eテストストーリー

## テスト対象API
- basicInfo.yml
- authInfo.yml
- env.yml

## ストーリー

### 1. 基本設定情報の取得と更新
- [ ] **GET /basic-info** - 基本設定情報を取得
  - ドメイン名、DNS設定、メール設定などの確認
- [ ] **PUT /basic-info** - 基本設定情報を更新
  - ドメイン名、送信元メールアドレス、返信先メールアドレスの設定

### 2. 認証情報の設定
- [ ] **GET /auth-info** - 認証情報を取得
  - コールバックURLの確認
- [ ] **PUT /auth-info** - 認証情報を更新
  - コールバックURLの設定

### 3. 外部IDプロバイダの設定
- [ ] **GET /identity-providers** - 外部IDプロバイダ情報を取得
  - Google認証設定の確認
- [ ] **PUT /identity-providers** - 外部IDプロバイダ情報を更新
  - Google認証の設定更新

### 4. サインイン設定
- [ ] **GET /sign-in-settings** - サインイン設定を取得
  - パスワードポリシー、MFA設定、デバイス記憶設定の確認
- [ ] **PUT /sign-in-settings** - サインイン設定を更新
  - パスワードポリシーの更新

### 5. 環境情報の管理
- [ ] **GET /envs** - 環境一覧を取得
  - 登録済み環境の確認
- [ ] **POST /envs** - 新しい環境を作成
  - テスト用環境の作成
- [ ] **GET /envs/{env_id}** - 特定環境の詳細を取得
  - 作成した環境の詳細確認
- [ ] **PATCH /envs/{env_id}** - 環境情報を更新
  - 環境名や表示名の更新
- [ ] **DELETE /envs/{env_id}** - 環境を削除
  - テスト用環境の削除

### 6. カスタマイズページ設定
- [ ] **GET /customize-page-settings** - ページカスタマイズ設定を取得
  - アイコン、ファビコン、サービス名などの確認
- [ ] **PATCH /customize-page-settings** - ページカスタマイズ設定を更新
  - サービス名、利用規約URL、プライバシーポリシーURLの設定

### 7. 認証画面設定
- [ ] **GET /customize-pages** - 認証画面設定を取得
  - サインアップ、サインイン、パスワードリセット画面の設定確認
- [ ] **PATCH /customize-pages** - 認証画面設定を更新
  - 各画面の利用規約・プライバシーポリシー表示設定

### 8. 通知メールテンプレート
- [ ] **GET /notification-messages** - 通知メールテンプレートを取得
  - 各種通知メールの件名・本文確認
- [ ] **PUT /notification-messages** - 通知メールテンプレートを更新
  - サインアップ、パスワードリセット等のメールテンプレート更新

## 前提条件
- SaaSus Platformのアカウントが作成済み
- APIキーが発行済み
- テスト用のドメインが準備済み

## 期待結果
- 基本的な認証設定が完了している
- 環境が適切に設定されている
- カスタマイズ設定が反映されている
- 通知メールテンプレートが設定されている
