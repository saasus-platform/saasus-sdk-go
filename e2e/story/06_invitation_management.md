# 06. 招待管理ストーリー

## 概要
テナントへのユーザー招待機能のE2Eテストストーリー

## テスト対象API
- invitation.yml

## ストーリー

### 1. テナント招待の管理
- [ ] **GET /tenants/{tenant_id}/invitations** - テナント招待一覧を取得
  - 特定テナントの招待一覧取得
  - 招待ステータス（pending, accepted, expired）の確認
  - 招待URL、有効期限の確認
- [ ] **POST /tenants/{tenant_id}/invitations** - テナント招待を作成
  - 新規ユーザーの招待作成
  - 環境毎の役割指定
  - 招待メール送信
- [ ] **GET /tenants/{tenant_id}/invitations/{invitation_id}** - 招待情報を取得
  - 特定招待の詳細情報取得
  - 招待されたユーザーのメールアドレス確認
  - 環境・役割情報の確認
- [ ] **DELETE /tenants/{tenant_id}/invitations/{invitation_id}** - 招待を削除
  - 未承認招待の削除
  - 招待の取り消し

### 2. 招待の有効性確認
- [ ] **GET /invitations/{invitation_id}/validity** - 招待有効性を取得
  - 招待が有効かどうかの確認
  - 期限切れ招待の検証
- [ ] **PATCH /invitations/{invitation_id}/validate** - 招待を検証
  - 既存ユーザーの場合：アクセストークンで検証
  - 新規ユーザーの場合：メールアドレスとパスワードで検証

## テストシナリオ例

### シナリオ1: 新規ユーザーの招待フロー
1. テナントに新規ユーザーを招待
2. 招待一覧で招待が作成されていることを確認
3. 招待の有効性を確認
4. 新規ユーザーとして招待を検証（メール・パスワード）
5. 招待ステータスがacceptedに変更されることを確認
6. 招待されたユーザーがテナントに追加されることを確認

### シナリオ2: 既存ユーザーの招待フロー
1. 既存ユーザーを別のテナントに招待
2. 招待情報の詳細を確認
3. 既存ユーザーのアクセストークンで招待を検証
4. ユーザーが複数テナントに所属することを確認
5. 各テナントでの役割が正しく設定されることを確認

### シナリオ3: 招待の管理
1. 複数のユーザーを招待
2. 招待一覧で全ての招待を確認
3. 一部の招待を削除
4. 削除された招待が一覧から消えることを確認
5. 残った招待の有効性を確認

### シナリオ4: 招待の期限管理
1. 招待を作成
2. 招待の有効期限を確認
3. 期限切れ前の招待有効性確認
4. 期限切れ後の招待有効性確認（期限切れの場合）
5. 期限切れ招待の検証エラー確認

### シナリオ5: 環境・役割指定招待
1. 複数環境で異なる役割を指定して招待
2. 招待情報で環境・役割設定を確認
3. 招待を検証
4. ユーザーに指定した環境・役割が付与されることを確認
5. 環境毎の権限が正しく設定されることを確認

## 前提条件
- テナントが作成されている
- 役割(ロール)が定義されている
- 環境が設定されている
- 招待を作成するユーザーのアクセストークンが取得済み
- メール送信機能が設定されている

## 期待結果
- 招待が正常に作成できる
- 招待メールが送信される
- 招待の有効性確認が機能する
- 新規ユーザー・既存ユーザー両方の招待検証が動作する
- 招待されたユーザーが適切な権限でテナントに追加される
- 招待の削除・管理が正常に動作する

## 注意事項
- 招待の有効期限は設定により異なる
- 招待検証時のパラメータは新規/既存ユーザーで異なる
- 招待削除は未承認の招待のみ可能
- 招待メールの送信には適切なメール設定が必要
